# Banking Fraud Detection SQL Analysis

## Overview
This project focuses on analyzing banking transactions to detect fraudulent activities using SQL.

## Dataset
- Transaction ID
- Customer ID
- Transaction Date
- Amount
- Transaction Type
- Location
- Device Type
- Fraud Flag

## Key KPIs
- Total Transactions
- Fraud Percentage
- High Value Transactions
- Fraud by Location & Device
- Customer Risk Ranking

## SQL Concepts Used
- Aggregations (SUM, COUNT)
- CASE WHEN
- GROUP BY
- Window Functions (RANK, LAG)

## Key Insights
- High-value transactions are more likely to be fraudulent  
- Mobile-based transactions show higher fraud activity  
- Certain customers repeatedly perform suspicious transactions  
- Fraud concentration is higher in specific locations
